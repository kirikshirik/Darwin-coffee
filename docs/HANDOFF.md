# Darwin Coffee — Handoff (читать первым)

> Этот документ — точка входа для разработчика или AI-модели, которая продолжает проект
> с нуля. Прочитав его, ты должен понимать: что строим, что уже готово, что делать дальше
> и какие есть подводные камни. Детали — в соседних файлах [docs/](.).

---

## 1. Что это за продукт

**AI-аналитика для кофейни, которая показывает РЕАЛЬНУЮ ПРИБЫЛЬ, а не выручку.**

Первый клиент — реально работающая кофейня **«Дарвин»**. Касса — **Эвотор**.

Позиционирование (важно держать в голове при любом решении):
> «Эвотор показывает, на сколько ты продал. Наш сервис показывает, сколько ты реально
> заработал после всех расходов».

Выручку владелец видит в Эвоторе за 10 секунд. Чистая прибыль с учётом аренды, зарплат,
налогов и себестоимости — нигде автоматически не считается. Это и есть ценность.

Конечный вид MVP — Telegram-бот, который каждое утро присылает: выручка, чистая прибыль,
маржа, средний чек, топ товаров, прогноз прибыли месяца.

## 2. Текущий статус

### ✅ Сделано и ПРОВЕРЕНО
- Python-бэкенд (`backend/`) на SQLAlchemy 2.0.
- Модели данных под реальную кофейню (см. [ARCHITECTURE.md](ARCHITECTURE.md)).
- Реальный P&L «Дарвина» за 12 мес. перенесён из Excel владельца в код (`backend/darwin_data.py`).
- `ProfitCalculator` — расчёт прибыли + авто-проверки качества данных.
- **Контроль пройден:** калькулятор воспроизводит итоги Excel копейка-в-копейку
  (выручка 3 733 684 ₽, чистая прибыль 899 565 ₽). Команда проверки — в разделе 5.
- **Справочник себестоимости напитков** (`backend/cost_reference.py`) — из реального
  Excel фуд-коста владельца. Себестоимость позиций 250/350/450 мл сверена с контролем
  (67.82 / 77.45 / 79.24 ₽) → ✅. Плюс аудит закупочных цен (`cost_demo.py`).
- **Интеграция с Эвотором — каркас** (`backend/integrations/evotor/`): `EvotorClient`
  (async httpx) + маппинг ответа → `Product`/`Receipt`/`ReceiptItem` + апсерт в БД.
  **Эндпоинты и авторизация сверены с офиц. докой** (блокер №2 снят). Проверено offline
  на образцах: `python -m backend.integrations.evotor.demo`. Ждёт Cloud Token (блокер №1).
- **Telegram-бот + планировщик — каркас** (`backend/bot/`): aiogram 3, `/start` + кнопки
  (Сегодня/Вчера/Неделя/Месяц/Товары/Расходы/Прогноз/Аналитика), APScheduler шлёт утреннюю
  сводку. Вся арифметика — через `ProfitCalculator`. Проверено offline:
  `python -m backend.bot.demo` (годовой честный итог сходится с `honest_report` = 391 520 ₽).
- **Прогноз + «вау»-аналитика** (`backend/analytics/`): прогноз прибыли месяца (run-rate по
  чекам + историческая модель на накопленных месяцах — работает уже сейчас); топ по прибыли,
  прибыльные часы, неделя-к-неделе, рейтинг бариста. Проверено: `python -m backend.analytics.demo`.
- **Интерактивный сценарный калькулятор «Что-Если» (веб-панель):** полноценное моделирование
  влияния трафика, цен, аренды, ФОТ и удорожания сырья на чистую прибыль и безубыточность (break-even).
  Работает клиент-сайд (без задержек), развернут на Render: [https://darwin-bot.onrender.com/dashboard](https://darwin-bot.onrender.com/dashboard?key=26d4JUdiyevwBrZ6sWT6FV5nusEnGY7A).
- **Фоновый запуск синхронизации Эвотора (`POST /api/sync`):** переведен на запуск через
  изолированные асинхронные подпроцессы, что исключило конфликты Event Loop на сервере.

### ⏳ НЕ сделано (по приоритету — см. [ROADMAP.md](ROADMAP.md))
- **Подключить реальные данные:** Cloud Token Эвотора → первые ~100 чеков (сверить
  единицы денег и поля); токен бота от @BotFather → запустить `backend.bot.main`.
- Цены продажи по позициям → маржа по каждому товару (себестоимость уже есть).
- БД-таблицы рецептов (Ingredient/Recipe) — отложены до появления каталога товаров
  из Эвотора (сейчас себестоимость живёт в `cost_reference.py`, см. ARCHITECTURE §6).
- Рейтинг бариста: резолв `user_id`→имя через employees-эндпоинт Эвотора (сейчас пишем raw).
- Alembic-миграции (пока схема пересоздаётся в `seed.py`).
- Тесты (`pytest`).

## 3. Стек и ключевые решения

| Что | Выбор | Почему |
|---|---|---|
| Язык | Python 3.9+ | План проекта на Python; богатая экосистема для аналитики/прогнозов |
| ORM | SQLAlchemy 2.0 (синхронный) | Стандарт; легко сменить БД без правок моделей |
| БД (локально) | SQLite (`darwin.db`) | Стартуем без Docker/Postgres |
| БД (прод) | PostgreSQL | Переключается через `DATABASE_URL`, код не меняется |
| Деньги | `Decimal` / `Numeric(12,2)` | Без float-погрешностей |
| Бот | aiogram 3 | каркас готов (`backend/bot/`) |
| Планировщик | APScheduler 3 | утренняя сводка (`backend/bot/scheduler.py`) |
| HTTP | httpx (async) | `EvotorClient` |

> **Был Node.js/TypeScript-каркас** (`src/`, `package.json`, `node_modules/`). Проект
> переведён на Python; эти файлы больше не используются и могут быть удалены.

## 5. Как запустить с нуля

```bash
# 1. Окружение
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt

# 2. Наполнить БД реальными данными «Дарвина»
.venv/bin/python -m backend.seed

# 3. Отчёт P&L + автоматическая сверка с Excel
.venv/bin/python -m backend.report_demo

# 4. Себестоимость напитков + аудит цен + проверка Находки 1 (без БД)
.venv/bin/python -m backend.cost_demo

# 5. Честный P&L (реальный ФОТ + COGS) — Находка 2
.venv/bin/python -m backend.honest_report

# 6. Интеграция с Эвотором — offline-проверка маппинга (без Cloud Token)
.venv/bin/python -m backend.integrations.evotor.demo

# 7. Telegram-бот — offline-проверка отчётов (без токена бота)
.venv/bin/python -m backend.bot.demo

# 8. Прогноз + «вау»-аналитика (Фаза 4) — offline на образцах
.venv/bin/python -m backend.analytics.demo

# 9. Запустить бота (нужен TELEGRAM_BOT_TOKEN в .env, см. .env.example)
.venv/bin/python -m backend.bot.main
```

Ожидаемый результат шага 3 — таблица P&L по месяцам и строка `=> ✅ СОВПАДАЕТ`.
Шаг 4 — раскладка себестоимости 250/350/450 мл со строкой `✅ ВСЕ ПОЗИЦИИ СОВПАДАЮТ`.
Шаги 6–8 — строки `✅ МАППИНГ РАБОТАЕТ`, `✅ Конвейер отчётов работает` (годовой честный
итог = 391 520 ₽, сходится с шагом 5) и `✅ Прогноз и аналитика работают`. Шаги 6–9 требуют
`pip install` из шага 1 (httpx/aiogram/apscheduler). Если расхождение — что-то сломано в
данных/калькуляторе/маппинге.

Сменить БД на PostgreSQL:
```bash
export DATABASE_URL="postgresql+psycopg://user:pass@host:5432/darwin"
# и добавить psycopg в requirements.txt
```

## 6. Главные находки по данным (бизнес-суть продукта)

Калькулятор не просто считает — он **вскрывает ошибки в учёте владельца**. Две находки
из реальных данных, и обе автоматически помечаются флагами в `ProfitCalculator`:

1. **Себестоимости (COGS) нет отдельной статьёй** — она спрятана в «Прочее»
   (~28% выручки). Теперь есть справочник рецептов (`cost_reference.py`): себестоимость
   чашки ≈ 68–79 ₽, что при фуд-косте 28.7% даёт средний чек ≈ 261 ₽ — реалистично,
   т.е. «Прочее» **действительно ведёт себя как закупка товара** (подтверждено).

2. **ФОТ (зарплаты) не внесён в 6 месяцах из 12.** Поднят реальный ФОТ (дневной отчёт
   окт–май + сентябрь 80 600 ₽ от владельца) → честная прибыль **391 520 ₽/год (43.5%)**
   против показанных в Excel 899 565 ₽. Считает `backend/honest_report.py`.
   **Это сильнейший аргумент в продаже продукта** — Excel врёт, наш сервис покажет правду.

3. **Несостыковки закупочных цен** (аудит в `cost_demo.py`): молоко и сахар сходятся,
   а кофе владелец округляет (37.40→38.00 ₽), корица заявлена 40 ₽/8 г (=2.50 ₽/порцию),
   но в расчёте стоит 0.80 ₽ — расхождение, требует уточнения у владельца.

Подробно с цифрами — в [DATA.md](DATA.md).

## 7. Открытые вопросы и блокеры

Каркасы интеграции и бота готовы; остались внешние секреты и валидация на реальных данных:

- **Cloud Token Эвотора** (🔴) для аккаунта «Дарвина» — без него API недоступен.
  Положить в `.env` как `EVOTOR_CLOUD_TOKEN` (см. `.env.example`).
- **Токен Telegram-бота** (🔴) от @BotFather + `TELEGRAM_OWNER_CHAT_ID` владельца.
- ~~**Эндпоинты Эвотора — догадка из `info`**~~ ✅ снято: сверены с https://developer.evotor.ru
  (`X-Authorization` raw, `/stores`, `/stores/{id}/products`, `/stores/{id}/documents`,
  `since`/`until` в мс, media-type v2). Реальная схема — в шапке `evotor/client.py`.
- **Единицы денег в чеках** (🟡) — рубли или копейки; подтвердить на первых реальных
  чеках и при необходимости включить `EVOTOR_MONEY_IN_KOPECKS` (`evotor/mapping.py`).
- **Цены продажи по позициям** (🔴) — пока неизвестны. Себестоимость есть (`cost_reference.py`),
  но для маржи по каждому товару нужны цены продажи (получить у владельца).
- **РАФ и часть допов** (сливки, арахисовая паста) — в Excel себестоимость порции не
  заполнена, поэтому в расчёт не берутся (`ADDONS_NO_COST`).

Вопросы владельцу (валидация) — в [ROADMAP.md](ROADMAP.md).

## 8. Правила для того, кто продолжает (do / don't)

- **Источник правды по данным — `backend/darwin_data.py` и Excel-файлы.** Файл `info` —
  это история размышлений, в нём есть устаревшие и ошибочные куски (кривой Python с
  опечатками, выдуманные эндпоинты). Не копировать оттуда код вслепую.
- **Пустая ячейка ≠ ноль.** В данных пропуск означает «не внесено», а не «расход = 0».
  Модель данных и калькулятор это различают — сохраняй это поведение.
- **Любое изменение модели/калькулятора проверяй командой из раздела 5** — итог должен
  оставаться `✅ СОВПАДАЕТ` с Excel.
- **Деньги — только `Decimal`**, не `float`.
- Сначала ценность для владельца, потом красота. MVP важнее идеального продукта.
