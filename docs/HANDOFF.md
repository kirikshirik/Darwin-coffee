## 1. Что это за продукт

**AI-аналитика для кофейни, которая показывает РЕАЛЬНУЮ ПРИБЫЛЬ, а не выручку.**

Первый клиент — реально работающая кофейня **«Дарвин»**. Касса — **Эвотор**.

Позиционирование (важно держать в голове при любом решении):
> «Эвотор показывает, на сколько ты продал. Наш сервис показывает, сколько ты реально
> заработал после всех расходов».

Выручку владелец видит в Эвоторе за 10 секунд. Чистая прибыль с учётом аренды, зарплат,
налогов и себестоимости — нигде автоматически не считается. Это и есть ценность.

Конечный вид MVP — Telegram-бот, который каждое утро присылает: выручка, чистая прибыль,
маржа, средний чек, топ товаров, прогноз прибыли месяца.

## 2. Текущий статус

### ✅ Сделано и ПРОВЕРЕНО
- Python-бэкенд (`backend/`) на SQLAlchemy 2.0.
- Модели данных под реальную кофейню (см. [ARCHITECTURE.md](ARCHITECTURE.md)).
- Реальный P&L «Дарвина» за 12 мес. перенесён из Excel владельца в код (`backend/darwin_data.py`).
- `ProfitCalculator` — расчёт прибыли + авто-проверки качества данных.
- **Контроль пройден:** калькулятор воспроизводит итоги Excel копейка-в-копейку
  (выручка 3 733 684 ₽, чистая прибыль 899 565 ₽). Команда проверки — в разделе 5.

### ⏳ НЕ сделано (по приоритету — см. [ROADMAP.md](ROADMAP.md))
- `EvotorClient` — загрузка выручки/чеков/товаров из API Эвотора.
- Справочник себестоимости (рецепты) → маржа по товарам.
- Telegram-бот + планировщик ежедневных отчётов.
- Прогноз прибыли месяца.
- Alembic-миграции (пока схема пересоздаётся в `seed.py`).
- Тесты (`pytest`).

## 3. Стек и ключевые решения

| Что | Выбор | Почему |
|---|---|---|
| Язык | Python 3.9+ | План проекта на Python; богатая экосистема для аналитики/прогнозов |
| ORM | SQLAlchemy 2.0 (синхронный) | Стандарт; легко сменить БД без правок моделей |
| БД (локально) | SQLite (`darwin.db`) | Стартуем без Docker/Postgres |
| БД (прод) | PostgreSQL | Переключается через `DATABASE_URL`, код не меняется |
| Деньги | `Decimal` / `Numeric(12,2)` | Без float-погрешностей |
| Бот (план) | aiogram 3 | — |
| HTTP (план) | httpx (async) | — |

> **Был Node.js/TypeScript-каркас** (`src/`, `package.json`, `node_modules/`). Проект
> переведён на Python; эти файлы больше не используются и могут быть удалены.

## 5. Как запустить с нуля

```bash
# 1. Окружение
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt

# 2. Наполнить БД реальными данными «Дарвина»
.venv/bin/python -m backend.seed

# 3. Отчёт P&L + автоматическая сверка с Excel
.venv/bin/python -m backend.report_demo
```

Ожидаемый результат шага 3 — таблица P&L по месяцам и строка `=> ✅ СОВПАДАЕТ`.
Если расхождение — что-то сломано в моделях/данных/калькуляторе.

Сменить БД на PostgreSQL:
```bash
export DATABASE_URL="postgresql+psycopg://user:pass@host:5432/darwin"
# и добавить psycopg в requirements.txt
```

## 6. Главные находки по данным (бизнес-суть продукта)

Калькулятор не просто считает — он **вскрывает ошибки в учёте владельца**. Две находки
из реальных данных, и обе автоматически помечаются флагами в `ProfitCalculator`:

1. **Себестоимости (COGS) нет отдельной статьёй** — она спрятана в «Прочее»
   (~28% выручки). Следствие: маржу по товарам (капучино vs чизкейк) посчитать
   НЕЛЬЗЯ без собственного справочника себестоимости/рецептов.

2. **ФОТ (зарплаты) не внесён в 6 месяцах из 12.** Из-за этого Excel завышает прибыль
   почти вдвое: показано 899 565 ₽/год, реально ≈ 385 000 ₽/год.
   **Это сильнейший аргумент в продаже продукта** — Excel врёт, наш сервис покажет правду.

Подробно с цифрами — в [DATA.md](DATA.md).

## 7. Открытые вопросы и блокеры

Прежде чем писать `EvotorClient`, нужно снять блокеры:

- **Cloud Token Эвотора** для аккаунта «Дарвина» — есть ли доступ? Без него API-путь не стартует.
- **Эндпоинты Эвотора в файле `info` — это ДОГАДКА**, их нельзя брать на веру. Сверять
  с официальной докой: https://developer.evotor.ru
- **Себестоимость напитков** — данных нет ни в Эвоторе, ни в Excel. Нужен ручной справочник
  (рецепты: зерно/молоко/стакан/крышка). Без него п.2 роадмапа невозможен.

Вопросы владельцу (валидация) — в [ROADMAP.md](ROADMAP.md).

## 8. Правила для того, кто продолжает (do / don't)

- **Источник правды по данным — `backend/darwin_data.py` и Excel-файлы.** Файл `info` —
  это история размышлений, в нём есть устаревшие и ошибочные куски (кривой Python с
  опечатками, выдуманные эндпоинты). Не копировать оттуда код вслепую.
- **Пустая ячейка ≠ ноль.** В данных пропуск означает «не внесено», а не «расход = 0».
  Модель данных и калькулятор это различают — сохраняй это поведение.
- **Любое изменение модели/калькулятора проверяй командой из раздела 5** — итог должен
  оставаться `✅ СОВПАДАЕТ` с Excel.
- **Деньги — только `Decimal`**, не `float`.
- Сначала ценность для владельца, потом красота. MVP важнее идеального продукта.
