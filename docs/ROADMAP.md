## Фазы

### ✅ Фаза 0 — Ядро (СДЕЛАНО)
Модели данных, реальный P&L «Дарвина», `ProfitCalculator`, сверка с Excel.

### ✅ Фаза 2 — Справочник себестоимости (СДЕЛАНО, как модуль)
- `backend/cost_reference.py` — себестоимость напитков из реального Excel фуд-коста,
  сверена с контролем (67.82/77.45/79.24 ₽) + аудит закупочных цен.
- Закрыла Находку 1: подтверждено, что «Прочее» ведёт себя как закупка товара (COGS).
- **Осталось для полной маржи по товарам:** цены продажи по позициям (у владельца) +
  материализация в БД-таблицы (Ingredient/Recipe), когда Эвотор даст каталог товаров.
  Почему пока модуль, а не таблицы — см. [ARCHITECTURE.md](ARCHITECTURE.md) §6.

### ✅ Фаза 2.5 — Честный P&L на реальных данных (СДЕЛАНО)
- `backend/actuals_data.py` — реальный ФОТ + Food cost (COGS) из дневного отчёта «Отчет 25-26».
- `backend/honest_report.py` — наложение факта на P&L-Excel: честная прибыль **391 520 ₽**
  (43.5% от Excel-наивных 899 565 ₽) — Находка 2 закрыта реальными данными.
- Реальный COGS окт–май = 26.7% выручки → сошлось с рецептурной оценкой 28.7% (Находка 1).
- `MONTHLY` не тронут (контроль 899 565 ₽ цел) — факт лежит отдельным overlay.
- ФОТ закрыт полностью: окт–май — дневной отчёт, сен 25 — 80 600 ₽ от владельца, июн–авг — Excel.

### 🟡 Фаза 1 — Интеграция с Эвотором (КАРКАС ГОТОВ, ждёт Cloud Token)
- `backend/integrations/evotor/` — `EvotorClient` (async httpx):
  `get_stores()`, `get_products(store_id)`, `get_documents/get_sales(store_id, since, until)`.
- **Эндпоинты и авторизация СВЕРЕНЫ с developer.evotor.ru — блокер №2 снят.** Догадки из
  `info` были неверны; правильно: `X-Authorization: <токен>` (raw, без Bearer),
  `Accept: application/vnd.evotor.v2+json`, пути `/stores`, `/stores/{id}/products`,
  `/stores/{id}/documents` (`since`/`until` в мс, `type=SELL`, пагинация `cursor`).
- Маппинг Эвотор→`Product`/`Receipt`/`ReceiptItem` + идемпотентный апсерт (`mapping.py`).
- Проверено offline на образцах: `python -m backend.integrations.evotor.demo` (без токена).
- **Осталось (когда дадут Cloud Token):** реальные ~100 чеков → подтвердить единицы денег
  (рубли/копейки, `MONEY_IN_KOPECKS`), имена полей и доступность метода на тарифе.

### 🟡 Фаза 3 — Telegram-бот + планировщик (КАРКАС ГОТОВ, ждёт токен бота)
- `backend/bot/` — aiogram 3; `/start` + кнопки: Сегодня, Вчера, Неделя, Месяц, Товары, Расходы.
- Планировщик (APScheduler): каждое утро в `TELEGRAM_REPORT_TIME` шлёт сводку владельцу.
- Вся арифметика — через существующий `ProfitCalculator` (метрики в `bot/metrics.py`).
- «Месяц» уже сейчас показывает честный P&L из Excel (headline без Эвотора); дневные/недельные
  отчёты включатся автоматически, как только пойдут чеки из Эвотора.
- Проверено offline: `python -m backend.bot.demo` — годовой честный итог сходится с
  `honest_report` (391 520 ₽), сквозной путь Эвотор-образцы → БД → отчёт работает.
- **Осталось:** токен бота от @BotFather + chat_id владельца → запустить `backend.bot.main`.

### 🟢 Фаза 4 — Прогноз и «вау»-аналитика (СДЕЛАНО, `backend/analytics/`)
- **Прогноз месяца** (`forecast.py`), две модели: run-rate по чекам Эвотора (экстраполяция
  выручки/COGS, опер.расходы фиксированы) и **историческая** (среднее честной прибыли за
  N мес + «тот же месяц год назад») — историческая работает уже сейчас на реальных данных.
- **Инсайты** (`insights.py`): топ товаров по прибыли, самые прибыльные часы,
  сравнение неделя-к-неделе, рейтинг бариста (по `Receipt.cashier` = `user_id` Эвотора).
- Встроено в бота: кнопки «Прогноз» и «Аналитика» + прогноз прибыли в утренней сводке.
- Проверено offline: `python -m backend.analytics.demo` (на образцах за 2 недели).
- **Осталось (с реальными чеками):** рейтинг бариста резолвит `user_id`→имя через
  employees-эндпоинт Эвотора; модель прогноза можно усложнить (сезонность/тренд).

### Инфраструктурный долг (параллельно)

1. **Cloud Token Эвотора** → подключить реальные чеки (`EvotorClient` + `mapping.sync_*`),
   на первых ~100 чеках сверить единицы денег и поля.

   проверить утреннюю сводку.

Эндпоинты Эвотора уже сверены с офиц. докой (блокер №2 снят). Для маржи по товарам

(честная прибыль на его же данных) и провести **интервью** (вопросы ниже) — это дешевле,
чем ждать доступы.

## Открытые вопросы владельцу (валидация)

Прямо из находок по данным — это не абстракция, а конкретика:
1. Что входит в статью «Прочее»? Это закупка продуктов? (Находка 1 — ✅ подтверждено
   дневным отчётом: «Прочее» ≈ колонка Food cost; всё же уточнить, нет ли там не-товарных трат)
2. Почему ФОТ за март/май в P&L-Excel (80 600/80 400) расходится с дневным отчётом
   (82 647/82 424)? (Находка 2 — реальный ФОТ поднят, см. `honest_report.py`; ФОТ за
   сентябрь 80 600 ₽ получен от владельца, пробел закрыт)
3. Где сейчас смотрите прибыль (не выручку) по итогам дня? Как часто?
4. Цена корицы: в фуд-косте указано 40 ₽/8 г (=2.50 ₽/порцию), но в расчёте 0.80 ₽ —
   какая верна? (аудит `cost_demo.py`)
5. Цены продажи по позициям (250/350/450 мл и допы) — для расчёта маржи по товарам.
6. Есть ли доступ к личному кабинету Эвотора / Cloud Token?
7. За какой ежедневный отчёт были бы готовы платить и сколько (0 / 500 / 1000–3000 ₽/мес)?

Если за отчёт готовы платить ≥1000 ₽/мес — разработка оправдана.

## Definition of Done для MVP

Бот каждое утро присылает владельцу: выручка, чистая прибыль, маржа, средний чек,
топ-5 товаров, прогноз прибыли месяца — **автоматически, на реальных данных из Эвотора,
с честным учётом расходов** (включая ФОТ и себестоимость).
