## Фазы

### ✅ Фаза 0 — Ядро (СДЕЛАНО)
Модели данных, реальный P&L «Дарвина», `ProfitCalculator`, сверка с Excel.

### ⏳ Фаза 1 — Интеграция с Эвотором
- `backend/integrations/evotor/client.py` — `EvotorClient`:
  `get_stores()`, `get_products(store_uuid)`, `get_documents(store_uuid, date_from, date_to)`.
- httpx async, Bearer-токен в заголовке `Authorization`.
- Маппинг ответа Эвотора → модели `Product` / `Receipt` / `ReceiptItem`.
- **Блокеры:** см. раздел «Блокеры» ниже (токен + неподтверждённые эндпоинты).
- Цель этапа: получить первые ~100 реальных чеков и понять, какие поля реально приходят.

### ⏳ Фаза 2 — Справочник себестоимости
- Таблицы `recipes`, `recipe_items`, `suppliers` (см. [ARCHITECTURE.md](ARCHITECTURE.md) §6).
- Расчёт `product.cost_price` из рецепта и закупочных цен.
- Разблокирует честную **маржу по товарам** и заполняет `COGS` (закрывает Находку 1).

### ⏳ Фаза 3 — Telegram-бот + планировщик
- aiogram 3; команды: `/start`, Сегодня, Вчера, Неделя, Месяц, Товары, Расходы.
- Планировщик: каждое утро считает `DailyMetric` и шлёт сводку.
- Вся арифметика — через существующий `ProfitCalculator`.

### ⏳ Фаза 4 — Прогноз и «вау»-аналитика
- Прогноз выручки/прибыли месяца по накопленным данным.
- Топ товаров по прибыли, самые прибыльные часы, сравнение с прошлой неделей, рейтинг бариста.

### Инфраструктурный долг (параллельно)
- Alembic-миграции вместо drop/create в `seed.py`.
- PostgreSQL на проде (через `DATABASE_URL`).
- Тесты `pytest` (в первую очередь на `ProfitCalculator` и сверку с Excel).
- Удалить устаревший Node-каркас (`src/`, `package.json`, `node_modules/`).

## Ближайшая задача (рекомендация)

**Перед кодом `EvotorClient` — снять блокеры по Эвотору** (токен + сверка эндпоинтов).
Если токена ещё нет — параллельно двигать **Фазу 2 (справочник себестоимости)**, т.к. она
не зависит от API и закрывает главный пробел в данных.

Альтернатива по духу проекта «сначала ценность»: провести **интервью с владельцем**
(вопросы ниже) — это дешевле, чем неделя интеграции в неизвестность.

## Открытые вопросы владельцу (валидация)

Прямо из находок по данным — это не абстракция, а конкретика:
1. Что входит в статью «Прочее»? Это закупка продуктов? (проверка Находки 1)
2. Почему зарплаты не внесены за сен/ноя/дек/янв/фев/апр? Платятся налом / не учитываются?
   (проверка Находки 2)
3. Где сейчас смотрите прибыль (не выручку) по итогам дня? Как часто?
4. Ведёте ли себестоимость напитков? В каком виде?
5. Есть ли доступ к личному кабинету Эвотора / Cloud Token?
6. За какой ежедневный отчёт были бы готовы платить и сколько (0 / 500 / 1000–3000 ₽/мес)?

Если за отчёт готовы платить ≥1000 ₽/мес — разработка оправдана.

## Блокеры (снять до Фазы 1)

| Блокер | Деталь |
|---|---|
| **Cloud Token Эвотора** | нужен токен аккаунта «Дарвина»; без него API недоступен |
| **Эндпоинты Эвотора не подтверждены** | пути в файле `info` — догадка. Сверять с https://developer.evotor.ru |
| **Ограничения тарифа** | проверить, доступны ли документы/чеки через API на тарифе кофейни |
| **Нет данных по себестоимости** | ни в Эвоторе, ни в Excel; нужен ручной справочник рецептов |

## Definition of Done для MVP

Бот каждое утро присылает владельцу: выручка, чистая прибыль, маржа, средний чек,
топ-5 товаров, прогноз прибыли месяца — **автоматически, на реальных данных из Эвотора,
с честным учётом расходов** (включая ФОТ и себестоимость).
