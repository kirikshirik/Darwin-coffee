# Darwin Coffee

AI-аналитика для кофейни «Дарвин»: считает **реальную прибыль**, а не только выручку.
Подключается к Эвотору, учитывает расходы и себестоимость, шлёт ежедневный отчёт в Telegram.

## Стек

- Python 3.9+ · SQLAlchemy 2.0 (БД: SQLite локально → PostgreSQL на проде)
- Дальше: httpx (Эвотор API), aiogram (Telegram-бот), Alembic (миграции)

> Старый Node.js/TypeScript-каркас (`src/`, `package.json`) больше не используется — проект переведён на Python. Его можно удалить.

## Что уже есть

- `backend/models.py` — модели: Business, Product, Receipt, ReceiptItem, Expense, DailyMetric + перечень статей расходов `ExpenseCategory`
- `backend/darwin_data.py` — реальный P&L «Дарвина» за 12 мес. (из Excel владельца)
- `backend/financial/profit_calculator.py` — расчёт прибыли + проверки качества данных
- `backend/seed.py` — наполнение БД реальными данными
- `backend/report_demo.py` — отчёт + сверка с Excel

## Запуск

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt

.venv/bin/python -m backend.seed         # наполнить БД
.venv/bin/python -m backend.report_demo  # P&L + контроль против Excel
```

## Финансовая модель

```
Выручка
  − Себестоимость (COGS)   = Валовая прибыль
  − Операционные расходы    = Чистая прибыль
```

Калькулятор сверен с Excel: выручка 3 733 684 ₽, чистая прибыль 899 565 ₽ — совпадает.

## Известные пробелы в данных (вскрыты калькулятором)

1. **Себестоимости нет** — она спрятана в статье «Прочее» (~28% выручки). Без справочника себестоимости маржу по товарам не посчитать.
2. **ФОТ не внесён в 6 месяцах из 12** — из-за этого реальная годовая прибыль ≈ 385 000 ₽, а не 899 565 ₽.
