# Darwin Coffee

AI-аналитика для кофейни «Дарвин»: считает **реальную прибыль**, а не только выручку.
Подключается к Эвотору, учитывает расходы и себестоимость, шлёт ежедневный отчёт в Telegram.

## Стек

- Python 3.9+ · SQLAlchemy 2.0 (БД: SQLite локально → PostgreSQL на проде)
- Дальше: httpx (Эвотор API), aiogram (Telegram-бот), Alembic (миграции)

> Старый Node.js/TypeScript-каркас (`src/`, `package.json`) больше не используется — проект переведён на Python. Его можно удалить.

## Что уже есть

- `backend/models.py` — модели: Business, Product, Receipt, ReceiptItem, Expense, DailyMetric + перечень статей расходов `ExpenseCategory`
- `backend/darwin_data.py` — реальный P&L «Дарвина» за 12 мес. (из Excel владельца)
- `backend/financial/profit_calculator.py` — расчёт прибыли + проверки качества данных
- `backend/cost_reference.py` — справочник себестоимости напитков (рецепты, 250/350/450 мл + допы), сверен с Excel фуд-коста
- `backend/cost_demo.py` — раскладка себестоимости + проверка гипотезы «Прочее = закупка товара»
- `backend/actuals_data.py` — реальный ФОТ + Food cost (COGS) из дневного отчёта «Отчет 25-26»
- `backend/honest_report.py` — честный P&L: наложение реального ФОТ и COGS на P&L-Excel
- `backend/seed.py` — наполнение БД реальными данными
- `backend/report_demo.py` — отчёт + сверка с Excel

## Запуск

```bash
python3 -m venv .venv
.venv/bin/python -m pip install -r requirements.txt

.venv/bin/python -m backend.seed         # наполнить БД
.venv/bin/python -m backend.report_demo  # P&L + контроль против Excel
.venv/bin/python -m backend.cost_demo    # себестоимость напитков + проверка «Прочее = закупка»
.venv/bin/python -m backend.honest_report # честный P&L: реальный ФОТ + COGS из дневного отчёта
```

## Финансовая модель

```
Выручка
  − Себестоимость (COGS)   = Валовая прибыль
  − Операционные расходы    = Чистая прибыль
```

Калькулятор сверен с Excel: выручка 3 733 684 ₽, чистая прибыль 899 565 ₽ — совпадает.

## Известные пробелы в данных (вскрыты калькулятором)

1. **Себестоимости нет** — она спрятана в статье «Прочее» (~28% выручки). ✅ Закрыто: справочник рецептов (`cost_reference.py`, чашка ≈ 68–79 ₽) + реальный Food cost из дневного отчёта (`actuals_data.py`, **26.7% выручки** окт–май) — обе оценки сошлись, «Прочее» = закупка товара. Осталось завести цены продажи по позициям → маржа по каждому товару.
2. **ФОТ не внесён в 6 месяцах из 12** — ✅ поднят из дневного отчёта (`actuals_data.py`). Честная годовая прибыль = **334 651 ₽** против Excel-наивных 899 565 ₽. Пробелов по ФОТ больше нет (окт–май — дневной отчёт, сентябрь — от владельца). COGS свежезакрытого месяца, внесённый не полностью, заменяется реалистичной оценкой (`actuals_data.effective_food_cost`), чтобы прибыль не выглядела завышенной.
